ptcalc is an R package that implements the statistical methods defined in ISO 13528:2022 and ISO 17043:2024 for proficiency testing (PT) schemes. It gives PT coordinators and metrologists a set of pure, dependency-light functions for computing robust assigned values, evaluating participant performance through four score types, and assessing the homogeneity and stability of test items before a round begins.

## What problems ptcalc solves

Proficiency testing programs require calculations that go beyond what base R provides out of the box. Coordinators need outlier-resistant estimators that follow a specific iterative algorithm, score formulas that account for measurement uncertainty, and ANOVA-based homogeneity criteria that must match ISO text exactly. Writing these correctly from scratch is error-prone, and getting them wrong affects participant evaluations.

ptcalc packages these calculations as tested, documented functions so you can focus on running your PT scheme rather than re-implementing ISO formulas.

<Note>
  ptcalc was developed under contract OSE-282-3065-2025 for Universidad Nacional de Colombia (Laboratorio CALAIRE) and Instituto Nacional de Metrología. All calculations follow the normative text of the respective standards.
</Note>

## ISO standards implemented

ptcalc implements methods from two standards:

**ISO 13528:2022** — *Statistical methods for use in proficiency testing by interlaboratory comparison* — is the primary technical reference. ptcalc covers:
- Annex C.3 (Algorithm A, iterative Winsorization)
- Section 9.2 (homogeneity assessment)
- Section 9.3 (stability assessment)
- Section 9.4 (nIQR and MADe robust estimators)
- Sections 10.2–10.5 (z, z′, zeta, and En score calculations)

**ISO 17043:2024** — *Conformity assessment — General requirements for proficiency testing* — defines the overall framework for PT schemes. ptcalc's score evaluation thresholds and uncertainty propagation align with its requirements.

## Design principles

Every function in ptcalc follows a consistent set of rules:

- **Pure functions.** No side effects, no global state, no Shiny or GUI dependencies. Pass in data, get back a value or list.
- **NA propagation.** Invalid inputs (non-finite numbers, zero denominators, insufficient sample sizes) return `NA` or a list with an `error` field rather than throwing an error that interrupts your analysis.
- **Vectorized variants.** Score evaluation functions come in both scalar (`evaluate_z_score()`) and vectorized (`evaluate_z_score_vec()`) forms for use in data frames or `lapply` workflows.
- **Minimal dependencies.** ptcalc imports only `stats` (base R) and `dplyr`.

  <Accordion title="Score calculations (4 functions)">
    | Function | Formula |
    |---|---|
    | `calculate_z_score(x, x_pt, sigma_pt)` | z = (x − x_pt) / σ_pt |
    | `calculate_z_prime_score(x, x_pt, sigma_pt, u_xpt)` | z′ = (x − x_pt) / √(σ_pt² + u_xpt²) |
    | `calculate_zeta_score(x, x_pt, u_x, u_xpt)` | ζ = (x − x_pt) / √(u_x² + u_xpt²) |
    | `calculate_en_score(x, x_pt, U_x, U_xpt)` | En = (x − x_pt) / √(U_x² + U_xpt²) |
  </Accordion>
