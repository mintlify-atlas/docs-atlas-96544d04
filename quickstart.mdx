This quickstart walks you through the core ptcalc workflow from loading the library to classifying a participant's performance. By the end you will have called `run_algorithm_a()`, `calculate_z_score()`, and `evaluate_z_score()` — the three functions at the heart of most proficiency testing rounds.

## Prerequisites

Make sure you have ptcalc installed and loadable. If you have not done this yet, see the [installation guide](/installation).

## Step-by-step walkthrough

<Steps>
  <Step title="Load the library">
    Start every session by loading ptcalc:

    ```r
    library(ptcalc)
    ```

    All 21 exported functions are now available in your session.
  </Step>

  <Step title="Run Algorithm A to get the assigned value and robust SD">
    Algorithm A (ISO 13528:2022, Annex C.3) computes outlier-resistant estimates of location and scale through iterative Winsorization. Pass a numeric vector of participant results — including any outliers — and ptcalc handles convergence automatically.

    ```r
    # Six participant results; 50.0 is a clear outlier
    values <- c(10.1, 10.2, 9.9, 10.0, 10.3, 50.0)

    result <- run_algorithm_a(values)

    cat("Assigned value (x*):", result$assigned_value, "\n")
    cat("Robust SD (s*):     ", result$robust_sd, "\n")
    cat("Converged:          ", result$converged, "\n")
    cat("Participants used:  ", result$n_participants, "\n")
    ```

    Example output:

    ```
    Assigned value (x*): 10.1
    Robust SD (s*):       0.1414
    Converged:            TRUE
    Participants used:    6
    ```

    The outlier at 50.0 is Winsorized during each iteration, so the assigned value and robust SD reflect the central cluster rather than the extreme value.

    <Tip>
      You can inspect `result$winsorized_values` to see exactly which values were clamped during the final iteration, and `result$iterations` for the full convergence history.
    </Tip>
  </Step>

  <Step title="Calculate a z-score for a participant">
    Use the assigned value and robust SD from Algorithm A as the proficiency assessment parameters. Pass in a single participant's result to get their z-score:

    ```r
    # Participant submitted 10.5; assigned value and SD come from Algorithm A
    z <- calculate_z_score(
      x       = 10.5,
      x_pt    = result$assigned_value,
      sigma_pt = result$robust_sd
    )

    cat("z-score:", z, "\n")
    ```

    Example output:

    ```
    z-score: 2.829
    ```

    The z-score is (10.5 − 10.1) / 0.1414 ≈ 2.83. A result this far from the assigned value will fall into the questionable or unsatisfactory range.
  </Step>

  <Step title="Evaluate the z-score classification">
    `evaluate_z_score()` applies the ISO 13528 classification thresholds:
    - |z| ≤ 2 → **Satisfactorio** (Satisfactory)
    - 2 < |z| < 3 → **Cuestionable** (Questionable)
    - |z| ≥ 3 → **No satisfactorio** (Unsatisfactory)

    ```r
    classification <- evaluate_z_score(z)
    cat("Classification:", classification, "\n")
    ```

    Example output:

    ```
    Classification: Cuestionable
    ```

    <Note>
      `evaluate_z_score()` also accepts z′ (z-prime) and zeta scores — all three use the same ISO threshold criteria.
    </Note>
  </Step>

  <Step title="Evaluate multiple participants at once (optional)">
    For a full PT round you will have one result per participant. Use `evaluate_z_score_vec()` to classify all scores in a single call, which works naturally with data frames:

    ```r
    # Participant results from the round
    participant_results <- c(10.0, 10.1, 10.5, 9.7, 10.2, 11.0)

    # Compute z-scores for all participants using the same assigned value and SD
    z_scores <- sapply(
      participant_results,
      calculate_z_score,
      x_pt     = result$assigned_value,
      sigma_pt = result$robust_sd
    )

    # Classify all scores at once
    classifications <- evaluate_z_score_vec(z_scores)

    # View results as a data frame
    data.frame(
      result         = participant_results,
      z_score        = round(z_scores, 3),
      classification = classifications
    )
    ```

    Example output:

    ```
      result z_score   classification
    1   10.0  -0.707    Satisfactorio
    2   10.1   0.000    Satisfactorio
    3   10.5   2.829     Cuestionable
    4    9.7  -2.829     Cuestionable
    5   10.2   0.707    Satisfactorio
    6   11.0   6.364  No satisfactorio
    ```
  </Step>
</Steps>

## What you just did

In five steps you covered the core of a proficiency testing round:

1. Loaded ptcalc into your R session.
2. Used Algorithm A to derive a robust assigned value and standard deviation that resist the influence of outliers.
3. Computed a z-score for an individual participant result.
4. Classified the z-score against ISO 13528 thresholds.
5. Applied the same workflow to a full vector of participant results using the vectorized evaluation function.
